# コンバットフェイズ
アルカディア内でエイドロンとの戦闘を行うフェイズ。

## 0. 戦闘開始時
* **作戦判定**: 戦闘開始時、代表者1名が「ロジック」で判定を行う。成功すると、戦闘配置前にエネミーの配置を知ることができる。
* **戦闘配置**: 戦場は直線上の5エリアに分かれている。プレイヤーは1、2、3のいずれかのエリアにキャラクターを配置し、エネミーは3、4、5のエリアに配置される。

## 1. ラウンドの進行
戦闘はラウンド単位で管理され、以下のフェイズ順で進行する。

1. **セットアップフェイズ**: ラウンドの開始処理。各キャラクターはリソースダイスのロールを行う（初期値は7個）。
2. **イニシアティブフェイズ**: 未行動のキャラクターのうち、イニシアティブが最大のキャラクターを確認する。
3. **メインフェイズ**: イニシアティブフェイズで確認したキャラクターが手番を実行する。アクションスキルの使用や移動を行う。
   * ※全員が行動済みになるまで、イニシアティブフェイズとメインフェイズを繰り返す。
4. **クリンナップフェイズ**: ラウンドの終了処理。未使用のリソースダイスがあれば、2個まで次のラウンドに持ち越すことができる。

## 2. リソースダイスとコスト
リソースダイスはスキルを使用するためのコスト。スキル宣言時に、要求されるコスト（特定の出目など）と一致するダイスをストックから消費する。

コストは必要なリソースダイス1つにつき`[]`で表記され、`[*]`なら任意の出目、`[1]`なら1の出目、`[34]`なら3か4の出目が必要となる。

## 3. エネミーの行動
エネミーはリソースダイスを管理せず、固有の制限に従って行動する。

* **コスト免除**: エネミーはスキル使用時にリソースダイスのコスト支払いを必要としない。
* **アクション回数**: エネミーは自分の手番に、固有ステータスである「アクション回数」の分だけ「タイミング：アクション」のスキルを使用できる。

## 4. プレイヤーとエネミーの基本スキル
全てのプレイヤーキャラクターは、最初から使用可能な基本スキルを修得している。
同様に全てのエネミーが使用可能な基本スキルも存在する。

## 5. 状態
キャラクターはスキルやアイテムなどの効果により、特殊な状態を付与されることがある。

状態は、自動解除タイミングで自動的に状態が解除されるか、スキルやアイテムの効果で解除されるまで継続する。

ここに記載されている状態以外にも、他の状態が付与されることもある。その状態の詳細はそれを付与するスキルやアイテムの効果欄を参照。

| 状態           | 自動解除タイミング | 効果                                                                                                                                                 |
| -------------- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| 最適化         | クリンナップ       | あらゆる判定に+1の修正                                                                                                                               |
| 断片化         | クリンナップ       | あらゆる判定に-1の修正                                                                                                                               |
| リペアn        | 戦闘終了           | クリンナップフェイズにHPをn点回復                                                                                                                    |
| ウイルスn      | 戦闘終了           | クリンナップフェイズにHPをn点失う                                                                                                                    |
| シンクロ       | クリンナップ       | イニシアティブ+3                                                                                                                                     |
| ラグ           | 戦闘終了           | イニシアティブ-3                                                                                                                                     |
| キャッシュ     | クリンナップ       | 即座にランダムなリソースダイス2個をストックに加える。クリンナップフェイズにリソースダイスを2個選んで失う。エネミーの場合、アクション回数が+1される。 |
| オーバーヘッド | クリンナップ       | コストが存在するプラグインスキルのコストに`[*]`を追加する。エネミーの場合、アクション回数が-1される。                                                |

## 6. フェイタル判定
プレイヤーキャラクターのHPが0以下になったとき、ただちに生死をかけた【ウィル】による判定を行う。これを**フェイタル判定**と呼ぶ。

* **HP低下によるマイナス修正**: フェイタル判定はHPが-10を下回る10点ごとに、判定に-1の修正を受ける。（例：HPが0〜-9なら修正なし、-10〜-19なら修正-1、-20〜-29なら修正-2、……）。
* **戦闘続行**: フェイタル判定に成功した場合はそのまま戦闘を続行できるが、以降ダメージを受けてHPが減少するたびに、同様の判定を再度行わなければならない。
* **ロスト（死亡）**: フェイタル判定に失敗した場合、筐体からのフィードバックにより現実の身体の脳神経が焼き切れ、キャラクターはロストとなる。
